    git wiki

    手順

    リモート事前準備

    • 1.Github上でリポジトリを新規作成

    ローカルへの導入

    • 2.ローカルにて、cmdを起動し、作業ディレクトリに移動
    • 3.リポジトリからcloneでディレクトリが作成される
    git clone <repository URL>
git clone -b <branch name> <repository URL>
    • 4.作成されたディレクトリに移動
    cd <clone directory>

    ローカルでの変更の適用

    • 5.必要なファイルを追加、更新などの変更を加える
    • 6.GitHub上で管理したくないものについては、.gitignoreを作成し、記述する。

    Warning

    • API KEYやTOKENなどを記載したconfigなどは絶対にGitに追加しないよう.gitignoreに追加する事！
    • 一度pushするとさかのぼっての削除はできません！
    • 7.変更をadd
    git add .

    Note

    • git add . することで新規追加されたファイルは初めてgitで管理する対象に加わります。
    • 8.変更をcommit
    git commit -m "commit message"

    Warning

    • pushしていない状態のcommit段階であれば、取り消しは可能です。push前に要確認。

    リモートへ変更の反映

    • 9.リモートリポジトリに変更を反映
    git push

    その他コマンド

    リモートリポジトリでの変更をローカルに取得取得

    git pull

    git addなどの状態確認

    git status

    差分の確認

    git diff --cached
git diff --cached ファイル名

    git addの取り消し

    git reset

    private repositoryを共有するアカウントの追加

    • 該当のリポジトリを開く
    • タブからSettingを選択
    • 左のナビゲーションバーからCollaboratorsを選択
    • アカウント名を入力し、該当アカウントが表示されたら選択
    • Add collaboratorを選択
